# 특별재고 대시보드 v1.0 완성본 체크리스트

이 문서는 실제 업무용으로 사용하기 전 마지막으로 확인할 항목입니다.

## 1. 최신 파일 받기

- GitHub에서 프로젝트 전체를 ZIP으로 다운로드하거나 `git pull`로 최신 상태를 받습니다.
- 프로젝트 폴더 안에 `manage.py`, `requirements.txt`, `inventory/`, `special_stock/`가 있는지 확인합니다.

## 2. 실행 확인

Windows에서는 프로젝트 폴더에서 아래 파일을 실행합니다.

```bat
run_windows_cmd.bat
```

정상이라면 브라우저에서 아래 주소가 열립니다.

```text
http://localhost:8000
```

## 3. DB 업데이트

새 버전을 받은 뒤에는 반드시 마이그레이션이 한 번 실행되어야 합니다. `run_windows_cmd.bat`는 내부에서 자동으로 아래 명령을 실행합니다.

```bat
python manage.py migrate
```

## 4. 실제 파일 검수

아래 3개 파일을 실제 업무 파일로 업로드해 확인합니다.

1. 재고/판매 통합 파일
2. 상품기본정보/오픈일 파일
3. 이카운트 발주 다운로드 파일(입고예정)

확인 항목:

- 품목별 요약 수량이 기존 엑셀과 맞는지
- 품목별 상세 합계가 맞는지
- 이카운트 입고예정의 `일자-No.`, `품목코드`, `미구매수량`, `출고일`이 정상 반영되는지
- `미구매수량 <= 0` 행이 제외되는지
- 마감 체크/해제가 저장되는지
- 4주 이하/8주 이하 카드 목록에 마감 배지가 표시되는지

## 5. 운영 데이터 백업

실제 운영을 시작하면 아래 항목을 주기적으로 백업합니다.

```text
db.sqlite3
media/
```

## 6. v1.0 확정 기준

아래 조건을 만족하면 v1.0 완성본으로 보고 사용을 시작합니다.

- 실제 재고/판매 파일 업로드 성공
- 실제 오픈일 파일 업로드 성공
- 실제 이카운트 입고예정 파일 업로드 성공
- 대표 품목 5개 이상 기존 엑셀과 수치 비교 완료
- 입고일정 발주번호 그룹 수정/삭제 확인
- 마감 체크 저장 확인
