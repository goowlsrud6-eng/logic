# 특별재고 관리 시스템 요구사항

## 목표

엑셀 시트를 복사하고 VBA를 실행하는 방식이 아니라, 원본 파일을 업로드하면 옵션 단위 계산 → 품목 단위 집계 → 대표님 보고용 요약 대시보드가 자동 생성되는 웹 기반 특별재고 관리 시스템을 만든다.

## 원본 데이터

### 1. 재고/판매 통합 데이터

업로드 주기: 매주 또는 월/수/금 등 정기 업로드

필수 데이터:

- 공급처옵션
- 상품코드
- 상품명
- 옵션명
- 현재고
- 최근한주 판매수량
- 총판매수량
- 배송수량
- 미출고/접수수량

### 2. 상품기본정보/오픈일

업로드 주기: 신규 품목 발생 시 또는 오픈일 변경 시

필수 데이터:

- 품목명 또는 상품명
- 옵션명, 옵션 단위 관리가 필요한 경우
- 공급처옵션, 있으면 사용
- 오픈일

오픈일은 총판매 기준 판매가능주 계산에 사용한다.

### 3. 입고예정 데이터

업로드 주기: 변동 시

필수 데이터:

- 공급처옵션
- 상품명
- 옵션명
- 입고예정수량
- 입고예정일, 아직 일정 미정이면 비워둘 수 있음
- 비고

입고예정은 확정 재고가 아니라 일정이므로, 같은 공급처옵션/입고예정일이 다시 들어오면 수정하고 신규 일정이면 추가한다. 변경 히스토리는 가능하면 보관한다.

## 계산 순서

반드시 옵션 단위에서 먼저 계산하고, 이후 품목 단위로 집계한다.

```text
옵션 단위 데이터
↓
옵션 단위 계산
↓
품목 단위 집계
↓
요약 대시보드
```

처음부터 품목 기준으로 계산하면 기존 엑셀과 결과가 달라질 수 있다.

## 주요 계산식

### 입고후재고

```text
입고후재고 = 현재고 + 입고예정수량
```

### 최근한주 기준 판매가능주

```text
현재고 기준 = 현재고 / 최근한주 판매수량
입고포함 기준 = 입고후재고 / 최근한주 판매수량
```

### 총판매 기준 판매가능주

```text
판매기간 = 기준일 - 오픈일 + 1
일평균 판매량 = 총판매수량 / 판매기간
주간 판매량 = 일평균 판매량 * 7
현재고 기준 = 현재고 / 주간 판매량
입고포함 기준 = 입고후재고 / 주간 판매량
```

## 판매 상승/하락 판정

현재 주차의 `최근한주 기준 판매가능주(입고포함)`과 지난 주차의 `지난주 판매 기준 판매가능주(입고포함)`을 비교한다.

```text
판매가능주 감소 = 판매속도 증가 = 판매 상승
판매가능주 증가 = 판매속도 감소 = 판매 하락
```

판정 단계:

- 판매 상승
- 판매 급상승
- 판매 하락
- 판매 급하락

기준:

- 3배 이상 차이: 상승/하락
- 10배 이상 차이: 급상승/급하락

## 공급처옵션 자동채움

공급처옵션이 비어 있는 경우, 동일 상품 내 옵션명을 정규화한 뒤 기존 공급처옵션을 찾아 자동으로 채운다.

정규화 규칙:

1. `★1차`, `★2차`, `★3차` 등 차수 제거
2. 공백 제거
3. 특수문자 제거: `/ _ ( ) [ ] { } & , . -` 등
4. 영문 소문자 통일

예시:

```text
★2차 주니어&성인-투톤베이지
★2차 주니어&성인/투톤 베이지
```

둘 다 아래 값으로 정규화한다.

```text
주니어성인투톤베이지
```

업로드 결과에는 아래 통계를 표시한다.

- 공급처옵션 자동 채움 성공 건수
- 공급처옵션 자동 채움 실패 건수

## 주차 자동 계산

사용자가 `0615-0619`를 직접 입력하지 않도록 한다.

입력:

```text
기준일: 2026-06-15
```

자동 생성:

```text
0615-0619
```

월요일 기준 주차이며 월~금 범위를 사용한다.

## 화면 구성

### 1. 요약 페이지

모든 품목에 대한 대표님 보고용 요약 화면이다.

필요 KPI:

- 전체 품목 수
- 전체 옵션 수
- 4주 이하 품목 수
- 8주 이하 품목 수
- 판매 급상승 품목 수
- 판매 급하락 품목 수
- 입고예정 없는 품목 수
- 재고 부족 위험 품목 수

필요 표 컬럼:

- 품목명
- 현재고
- 입고예정
- 입고후재고
- 최근한주 판매
- 총판매
- 판매가능주
- 전주 대비 상태

### 2. 품목별 상세 페이지

요약 페이지에서 품목을 클릭하면 이동한다.

옵션 단위 상세를 보여준다.

필요 컬럼:

- 옵션명
- 현재고
- 입고예정일
- 입고예정수량
- 입고후재고
- 최근한주 판매
- 총판매
- 판매가능주
- 전주 대비 상태

### 3. 입고일정 페이지

입고예정 파일에서 업로드한 일정만 별도로 보여준다.

필요 컬럼:

- 공급처옵션
- 상품명
- 옵션명
- 입고예정일
- 입고예정수량
- 비고
- 등록일

## 업로드 방식

한 화면에서 아래 파일을 선택하고 업로드할 수 있어야 한다.

1. 재고/판매 통합 파일
2. 상품기본정보/오픈일 파일
3. 입고예정 파일

오픈일 파일과 입고예정 파일은 변경 시에만 업로드하고, 재고/판매 통합 파일은 주기적으로 업로드한다.

## 이번 구현에서 반드시 지켜야 할 데이터 저장 원칙

- 재고/판매 통합 파일, 상품기본정보/오픈일 파일, 입고예정 파일은 서로 다른 파일 종류로 저장한다.
- 입고예정 파일을 마지막에 업로드해도 기존 재고/판매 옵션 데이터는 삭제하거나 대체하지 않는다.
- 대시보드 계산은 마지막 재고/판매 통합 데이터에 현재 `예정` 상태인 입고예정 데이터를 결합해서 수행한다.
- 상품기본정보/오픈일은 총판매 기준 판매가능주 계산에만 참조하며, 재고/판매 데이터를 대체하지 않는다.

## 입고예정 상태 관리

입고예정 데이터는 아래 상태를 가진다.

- 예정: 대시보드 입고예정수량과 입고후재고 계산에 반영
- 완료: 집계 제외
- 취소: 집계 제외

입고예정일이 지났는데 상태가 `예정`이면 입고일정 화면에서 `입고지연`으로 표시한다.

## 입고예정 날짜 입력 및 직접 관리

입고예정일은 아래 입력을 모두 같은 날짜로 해석한다.

- `20260624`
- `2026/06/24`
- `2026-06-24`
- `6/24`
- `06/24`
- `0624`

연도가 없는 `6/24`, `06/24`, `0624`는 기준일의 연도를 사용한다. 입고예정 화면에서는 업로드 외에도 직접 추가, 수정, 삭제가 가능해야 한다. 같은 공급처옵션/상품/옵션에 여러 입고 차수가 있으면 비고로 구분해 여러 건을 유지하고 합산한다.

## 주차별 스냅샷 보관

재고/판매 통합 파일은 업로드할 때마다 새 주차 스냅샷으로 저장한다. 신규 주차를 올려도 이전 주차 데이터는 삭제하지 않는다. 대시보드는 기본적으로 최신 주차를 보여주고, 필요하면 과거 주차 스냅샷을 선택해 조회할 수 있어야 한다.

## 발주번호 기준 입고예정 그룹 관리

입고예정 데이터에는 `발주번호`를 저장한다. 같은 발주번호에 속한 여러 옵션 입고예정은 하나의 그룹으로 관리할 수 있어야 한다.

- 입고예정 엑셀 양식에 발주번호 컬럼 포함
- 발주번호별 옵션 수와 총 입고예정수량 표시
- 발주번호 기준 입고예정일 일괄 수정
- 발주번호 기준 비고 일괄 수정
- 발주번호 기준 전체 삭제
- 기존 옵션별 개별 수정/삭제 기능은 유지

예를 들어 `2026/05/28-1` 발주번호에 옵션 160개가 있으면, 입고예정일 변경 또는 발주 취소를 160번 반복하지 않고 발주번호 기준으로 한 번에 처리한다.

## 지난주 기준 계산 방식

`지난주 기준 판매가능주`는 직전 주차 스냅샷의 판매가능주 값을 그대로 복사하는 것이 아니라, 현재 주차의 입고후재고를 직전 주차의 최근한주 판매수량으로 나누어 계산한다.

```text
지난주 기준 판매가능주 = 현재 주차 입고후재고 / 직전 주차 최근한주 판매수량
```

직전 주차 스냅샷은 현재 조회 중인 주차의 기준일보다 이전인 가장 최신 재고/판매 스냅샷으로 찾는다. 매칭은 공급처옵션, 상품코드, 상품명+정규화 옵션명 순서로 가능한 키를 모두 사용해 시도한다.

## 품목 탐색 편의 기능

요약 페이지에는 상품명, 상품코드, 공급처옵션 일부 단어로 검색할 수 있는 검색창을 제공한다. 판매상태와 입고예정 유무로도 필터링할 수 있어야 한다.

품목 상세를 열면 마지막 조회 품목을 세션에 저장하고, 요약/입고일정/상세 화면에서 최근 조회 품목과 즐겨찾기 품목을 바로 열 수 있어야 한다. 상세 화면에는 이전 품목/다음 품목 이동 버튼을 제공한다.

## KPI 카드 드릴다운 개선

대시보드 상단의 다음 KPI 카드는 클릭 시 해당 품목 목록을 바로 확인할 수 있어야 합니다.

* 4주 이하
* 8주 이하
* 판매 급상승
* 판매 급하락

목록에는 품목명, 현재주 기준 판매가능주, 지난주 기준 판매가능주, 현재고, 입고예정, 입고후재고, 최근한주 판매, 판매상태를 표시하며, 품목명을 클릭하면 품목 상세 화면으로 이동합니다.

## 품목별 요약 컬럼 개선

대시보드 품목별 요약은 실제 특별재고 엑셀 검토 순서에 맞춰 다음 컬럼을 표시한다.

1. 품목명
2. 가용재고
3. 입고예정수량
4. 입고후재고
5. 배송
6. 미출고수량(접수)
7. 판매수량(지난주)
8. 판매수량(최근한주)
9. 일평균(최근한주)
10. 판매수량(총판매)
11. 일평균(총판매)
12. 총판매 기준 현재고 판매가능주
13. 총판매 기준 입고포함 판매가능주
14. 최근한주 기준 현재고 판매가능주
15. 최근한주 기준 입고포함 판매가능주

일평균(최근한주)은 판매수량(최근한주) ÷ Min(7, 오픈후경과일수)로 계산하고, 일평균(총판매)은 판매수량(총판매) ÷ 오픈후경과일수로 계산한다. 카드 드릴다운 목록도 동일한 검토 컬럼을 표시한다.

재고/판매 통합 파일에서 대시보드의 배송 값은 현재고 양식의 `송장+배송` 열, 즉 사용자가 말한 I열 값을 우선 사용한다. `송장+배송` 열이 비어 있으면 기존 `배송` 열을 보조값으로 사용한다.

## 신규 품목 최근한주 일평균 예외처리

최근한주 일평균은 기본적으로 `최근한주 판매수량 ÷ 7`로 계산하지만, 오픈한 지 7일이 되지 않은 신규 옵션은 판매속도 과소평가를 방지하기 위해 `최근한주 판매수량 ÷ Min(7, 오픈후경과일수)`로 계산한다. 오픈후경과일수는 기준일과 오픈일의 날짜 차이로 계산하되 최소 1일로 본다.

예를 들어 기준일이 6/19이고 오픈일이 6/17이면 오픈후경과일수는 2일이므로 최근한주 판매수량을 2로 나눈다. 기준일이 6/19이고 오픈일이 6/10이면 오픈후경과일수는 9일이므로 최근한주 일평균은 7로 나누고, 총판매 일평균은 9로 나눈다. 최근한주 기준 현재고/입고포함 판매가능주도 이 보정된 일평균에 7을 곱한 주간 판매속도를 기준으로 계산한다.

## 요약 테이블 헤더 가독성 개선

품목별 요약과 KPI 카드 드릴다운 테이블은 긴 컬럼명을 그대로 표시하지 않고 축약명과 2줄 헤더를 사용한다. 전체 의미는 브라우저 기본 툴팁(`title`)으로 확인할 수 있게 한다.

예: `판매수량(지난주)`는 `지난주`, `최근한주 기준 입고포함 판매가능주`는 `최근주 입고포함`으로 표시한다.

## 판매상태 표시 및 색상 규칙

품목 상세와 품목별 요약 모두 판매상태를 표시한다. 판매상태는 현재주 최근주 입고포함 판매가능주와 지난주 기준 판매가능주를 비교해 판단한다.

* 판매 급감: 지난주 대비 10배 이상 악화, 분홍색 음영 + 빨간색 글씨
* 판매 하락: 지난주 대비 3배 이상 악화, 빨간색 글씨
* 판매 상승: 지난주 대비 3배 이상 개선, 파란색 글씨
* 판매 급등: 지난주 대비 10배 이상 개선, 하늘색 음영 + 파란색 글씨

기존에 저장된 `판매 급상승`, `판매 급하락` 표현은 화면 표시 시 각각 `판매 급등`, `판매 급감`으로 정규화한다.

## 총판매 일평균 계산 기준

총판매 일평균은 `총판매수량 ÷ 오픈후경과일수`로 계산한다. 최근한주 일평균만 최대 7일 기준을 적용하고, 총판매 일평균은 오픈 후 실제 경과일 전체를 사용한다.

## 마감 옵션 제외 기준

요약 계산과 품목 상세 표시에서 `가용재고 <= 0`이고 `입고예정수량 <= 0`인 옵션은 마감된 옵션으로 보고 제외한다. 다만 가용재고가 0 또는 음수여도 입고예정수량이 있으면 계속 표시하고 계산에 포함한다.

이후 같은 옵션에 재고가 다시 생기거나 입고예정수량이 등록되면 자동으로 다시 요약 계산과 상세 화면에 포함한다.

## 판매상태 판단 근거 표시

품목별 요약과 KPI 카드 드릴다운에는 `지난주 판매가능주`를 표시한다. 판매상태는 `최근주 입고포함 판매가능주`와 `지난주 판매가능주`를 비교해 판단하므로, 사용자가 요약 화면에서 상승/하락 판단 근거를 바로 확인할 수 있어야 한다.

품목 요약의 지난주 판매가능주는 현재 품목의 입고후재고를 직전 주차 최근한주 판매수량 합계로 나누어 계산한다.

## 판매상태 3배/10배 기준 엄격 적용

판매상태는 현재 계산된 `최근주 입고포함 판매가능주`와 `지난주 판매가능주`의 3배/10배 기준을 만족할 때만 표시한다. 과거 업로드 시 저장된 판매상태 값은 현재 화면의 판단 결과를 대체하지 않는다.

예를 들어 지난주 12.7주, 현재 13.2주는 약 1.04배 차이이므로 판매 하락/급감으로 표시하지 않는다. 지난주 15.0주, 현재 16.9주도 약 1.13배 차이이므로 판매상태를 표시하지 않는다.

## 품목 상세 합계행 및 일평균 표시

품목별 상세 화면은 요약 화면과 용어를 맞춰 `현재고` 대신 `가용재고`로 표시한다. 옵션 행에는 `일평균(최근한주)`와 `일평균(총판매)`를 함께 표시한다.

상세 화면 하단에는 `합계` 행을 표시한다. 합계 행의 가용재고, 입고예정수량, 입고후재고, 배송, 접수, 판매수량(지난주), 판매수량(최근한주), 일평균(최근한주), 판매수량(총판매), 일평균(총판매)은 표시된 옵션 행 값을 단순 합산한다.

합계 행의 일평균은 판매수량을 7로 다시 나누지 않고 옵션별 일평균의 합계를 사용한다. 합계 행의 판매가능주는 합계 가용재고/입고후재고와 합계 일평균에 7을 곱한 주간 판매속도를 사용해 요약 화면과 동일하게 계산한다.

## 입고일정 발주번호 그룹 대표 품목 표시

입고일정의 발주번호별 그룹 관리에는 발주번호만 표시하지 않고 대표 품목명을 함께 표시한다. 하나의 발주번호에 한 품목만 있으면 해당 품목명을 표시하고, 여러 품목이 섞여 있으면 `대표 품목명 외 N건` 형태로 표시한다.

대표 품목명은 클릭 또는 펼침 형태로 동작하여 해당 발주번호에 포함된 상품명, 옵션명, 공급처옵션, 수량, 입고예정일, 비고를 바로 확인할 수 있어야 한다.

## 이카운트 발주 다운로드 양식 입고예정 업로드

입고예정 업로드는 별도 수기 양식뿐 아니라 이카운트 발주 다운로드 양식을 그대로 사용할 수 있어야 한다. 이카운트 파일은 2행을 헤더로 보고 3행부터 데이터를 읽되, 열 위치가 아니라 헤더명(`일자-NO.`, `품목코드`, `품목명`, `규격`, `적요`, `미구매수량`, `출고일`)으로 매핑한다.

이카운트 매핑은 `일자-NO.` → 발주번호, `품목코드` → 공급처옵션, `품목명` → 품목명, `규격` → 옵션명, `적요` → 비고, `미구매수량` → 입고예정수량, `출고일` → 입고예정일로 처리한다. `미구매수량`이 0 이하인 행은 실제 입고예정이 없는 건으로 보고 등록하지 않는다.

입고일정 메인 화면의 발주번호 그룹 관리와 입고예정 직접 추가 기능은 유지하되, 옵션 수가 많은 경우를 고려해 전체 개별 목록은 메인 화면에 길게 표시하지 않는다. 발주번호 또는 대표 품목명을 클릭하면 해당 발주번호의 상세 화면으로 이동하고, 그 화면에서 옵션별 입고예정 수정/삭제를 수행한다.

## 품목 마감 여부 표시

시즌 후반 관리를 위해 품목별 요약에서 사용자가 체크박스로 품목의 마감 여부를 저장할 수 있어야 한다. 마감으로 체크된 품목은 요약 테이블, 4주 이하/8주 이하 카드 목록, 판매 급등/급감 목록에서 제외하지 않고 `마감` 배지 또는 체크 표시를 함께 보여준다.

메인 대시보드 카드 제목은 기준을 명확히 하기 위해 `최근한주 기준 4주 이하`, `최근한주 기준 8주 이하`처럼 표시한다. 판매 급등/급감 카드는 현재주 최근주 입고포함 판매가능주와 지난주 판매가능주 비교 기준임을 작은 설명으로 함께 표시한다.

## 이카운트 헤더/출고일 형식 보정

이카운트 발주 파일의 `일자-NO.`, `일자-No.`처럼 대소문자나 하이픈/마침표 표기가 조금 다른 헤더도 같은 컬럼으로 인식한다. 헤더 비교 전 공백, 하이픈, 마침표, 괄호 등 구분 문자를 제거하고 소문자로 통일한다.

입고예정일로 사용하는 `출고일`은 엑셀 날짜 값뿐 아니라 `7/8`, `7/14`처럼 문자열로 입력된 값도 기준연도 기준 날짜로 변환해 저장한다.

## 완성본 UI 방향

완성본 대시보드는 Tabler Dashboard 스타일의 상단 탭 기반 관리자 화면으로 구성한다. 왼쪽 사이드 메뉴는 제거하고 상단 탭만 사용해 `대시보드`, `요약 페이지`, `입고 일정`, `데이터 업로드`로 이동한다.

메인 대시보드는 KPI 카드, 판매가능주 분포, 판매 급등/급감 TOP 10, 총판매 TOP 10, 최근한주 판매 TOP 10처럼 보고용 현황만 보여준다. KPI 카드를 클릭하면 기존처럼 조건에 해당하는 품목 목록을 바로 확인하고 품목 상세로 이동할 수 있어야 한다.

품목별 요약표, 품목 검색, 판매상태 필터, 입고예정 필터는 별도 `요약 페이지`에서 제공한다. 데이터 업로드, 기준일 설정, 주차별 스냅샷, 최근 업로드 이력은 별도 `데이터 업로드` 탭에서 관리한다.

색상은 Primary Blue, Light Blue, Slate Gray, Light Gray 계열을 중심으로 사용한다. 카드는 흰색 배경, 은은한 그림자, 왼쪽 포인트 라인을 사용하고, 빨강/노랑 계열은 실제 경고나 판매 급감 등 필요한 상황에만 제한적으로 사용한다.
