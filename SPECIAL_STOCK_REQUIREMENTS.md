# 특별재고 관리 시스템 요구사항

## 목표

엑셀 시트를 복사하고 VBA를 실행하는 방식이 아니라, 원본 파일을 업로드하면 옵션 단위 계산 → 품목 단위 집계 → 대표님 보고용 요약 대시보드가 자동 생성되는 웹 기반 특별재고 관리 시스템을 만든다.

## 원본 데이터

### 1. 재고/판매 통합 데이터

업로드 주기: 매주 또는 월/수/금 등 정기 업로드

필수 데이터:

- 공급처옵션
- 상품코드
- 상품명
- 옵션명
- 현재고
- 최근한주 판매수량
- 총판매수량
- 배송수량
- 미출고/접수수량

### 2. 상품기본정보/오픈일

업로드 주기: 신규 품목 발생 시 또는 오픈일 변경 시

필수 데이터:

- 품목명 또는 상품명
- 옵션명, 옵션 단위 관리가 필요한 경우
- 공급처옵션, 있으면 사용
- 오픈일

오픈일은 총판매 기준 판매가능주 계산에 사용한다.

### 3. 입고예정 데이터

업로드 주기: 변동 시

필수 데이터:

- 공급처옵션
- 상품명
- 옵션명
- 입고예정수량
- 입고예정일, 아직 일정 미정이면 비워둘 수 있음
- 비고

입고예정은 확정 재고가 아니라 일정이므로, 같은 공급처옵션/입고예정일이 다시 들어오면 수정하고 신규 일정이면 추가한다. 변경 히스토리는 가능하면 보관한다.

## 계산 순서

반드시 옵션 단위에서 먼저 계산하고, 이후 품목 단위로 집계한다.

```text
옵션 단위 데이터
↓
옵션 단위 계산
↓
품목 단위 집계
↓
요약 대시보드
```

처음부터 품목 기준으로 계산하면 기존 엑셀과 결과가 달라질 수 있다.

## 주요 계산식

### 입고후재고

```text
입고후재고 = 현재고 + 입고예정수량
```

### 최근한주 기준 판매가능주

```text
현재고 기준 = 현재고 / 최근한주 판매수량
입고포함 기준 = 입고후재고 / 최근한주 판매수량
```

### 총판매 기준 판매가능주

```text
판매기간 = 기준일 - 오픈일 + 1
일평균 판매량 = 총판매수량 / 판매기간
주간 판매량 = 일평균 판매량 * 7
현재고 기준 = 현재고 / 주간 판매량
입고포함 기준 = 입고후재고 / 주간 판매량
```

## 판매 상승/하락 판정

현재 주차의 `최근한주 기준 판매가능주(입고포함)`과 지난 주차의 `지난주 판매 기준 판매가능주(입고포함)`을 비교한다.

```text
판매가능주 감소 = 판매속도 증가 = 판매 상승
판매가능주 증가 = 판매속도 감소 = 판매 하락
```

판정 단계:

- 판매 상승
- 판매 급상승
- 판매 하락
- 판매 급하락

기준:

- 3배 이상 차이: 상승/하락
- 10배 이상 차이: 급상승/급하락

## 공급처옵션 자동채움

공급처옵션이 비어 있는 경우, 동일 상품 내 옵션명을 정규화한 뒤 기존 공급처옵션을 찾아 자동으로 채운다.

정규화 규칙:

1. `★1차`, `★2차`, `★3차` 등 차수 제거
2. 공백 제거
3. 특수문자 제거: `/ _ ( ) [ ] { } & , . -` 등
4. 영문 소문자 통일

예시:

```text
★2차 주니어&성인-투톤베이지
★2차 주니어&성인/투톤 베이지
```

둘 다 아래 값으로 정규화한다.

```text
주니어성인투톤베이지
```

업로드 결과에는 아래 통계를 표시한다.

- 공급처옵션 자동 채움 성공 건수
- 공급처옵션 자동 채움 실패 건수

## 주차 자동 계산

사용자가 `0615-0619`를 직접 입력하지 않도록 한다.

입력:

```text
기준일: 2026-06-15
```

자동 생성:

```text
0615-0619
```

월요일 기준 주차이며 월~금 범위를 사용한다.

## 화면 구성

### 1. 요약 페이지

모든 품목에 대한 대표님 보고용 요약 화면이다.

필요 KPI:

- 전체 품목 수
- 전체 옵션 수
- 4주 이하 품목 수
- 8주 이하 품목 수
- 판매 급상승 품목 수
- 판매 급하락 품목 수
- 입고예정 없는 품목 수
- 재고 부족 위험 품목 수

필요 표 컬럼:

- 품목명
- 현재고
- 입고예정
- 입고후재고
- 최근한주 판매
- 총판매
- 판매가능주
- 전주 대비 상태

### 2. 품목별 상세 페이지

요약 페이지에서 품목을 클릭하면 이동한다.

옵션 단위 상세를 보여준다.

필요 컬럼:

- 옵션명
- 현재고
- 입고예정일
- 입고예정수량
- 입고후재고
- 최근한주 판매
- 총판매
- 판매가능주
- 전주 대비 상태

### 3. 입고일정 페이지

입고예정 파일에서 업로드한 일정만 별도로 보여준다.

필요 컬럼:

- 공급처옵션
- 상품명
- 옵션명
- 입고예정일
- 입고예정수량
- 비고
- 등록일

## 업로드 방식

한 화면에서 아래 파일을 선택하고 업로드할 수 있어야 한다.

1. 재고/판매 통합 파일
2. 상품기본정보/오픈일 파일
3. 입고예정 파일

오픈일 파일과 입고예정 파일은 변경 시에만 업로드하고, 재고/판매 통합 파일은 주기적으로 업로드한다.

## 이번 구현에서 반드시 지켜야 할 데이터 저장 원칙

- 재고/판매 통합 파일, 상품기본정보/오픈일 파일, 입고예정 파일은 서로 다른 파일 종류로 저장한다.
- 입고예정 파일을 마지막에 업로드해도 기존 재고/판매 옵션 데이터는 삭제하거나 대체하지 않는다.
- 대시보드 계산은 마지막 재고/판매 통합 데이터에 현재 `예정` 상태인 입고예정 데이터를 결합해서 수행한다.
- 상품기본정보/오픈일은 총판매 기준 판매가능주 계산에만 참조하며, 재고/판매 데이터를 대체하지 않는다.

## 입고예정 상태 관리

입고예정 데이터는 아래 상태를 가진다.

- 예정: 대시보드 입고예정수량과 입고후재고 계산에 반영
- 완료: 집계 제외
- 취소: 집계 제외

입고예정일이 지났는데 상태가 `예정`이면 입고일정 화면에서 `입고지연`으로 표시한다.

## 입고예정 날짜 입력 및 직접 관리

입고예정일은 아래 입력을 모두 같은 날짜로 해석한다.

- `20260624`
- `2026/06/24`
- `2026-06-24`
- `6/24`
- `06/24`
- `0624`

연도가 없는 `6/24`, `06/24`, `0624`는 기준일의 연도를 사용한다. 입고예정 화면에서는 업로드 외에도 직접 추가, 수정, 삭제가 가능해야 한다. 같은 공급처옵션/상품/옵션에 여러 입고 차수가 있으면 비고로 구분해 여러 건을 유지하고 합산한다.

## 주차별 스냅샷 보관

재고/판매 통합 파일은 업로드할 때마다 새 주차 스냅샷으로 저장한다. 신규 주차를 올려도 이전 주차 데이터는 삭제하지 않는다. 대시보드는 기본적으로 최신 주차를 보여주고, 필요하면 과거 주차 스냅샷을 선택해 조회할 수 있어야 한다.

## 발주번호 기준 입고예정 그룹 관리

입고예정 데이터에는 `발주번호`를 저장한다. 같은 발주번호에 속한 여러 옵션 입고예정은 하나의 그룹으로 관리할 수 있어야 한다.

- 입고예정 엑셀 양식에 발주번호 컬럼 포함
- 발주번호별 옵션 수와 총 입고예정수량 표시
- 발주번호 기준 입고예정일 일괄 수정
- 발주번호 기준 비고 일괄 수정
- 발주번호 기준 전체 삭제
- 기존 옵션별 개별 수정/삭제 기능은 유지

예를 들어 `2026/05/28-1` 발주번호에 옵션 160개가 있으면, 입고예정일 변경 또는 발주 취소를 160번 반복하지 않고 발주번호 기준으로 한 번에 처리한다.
