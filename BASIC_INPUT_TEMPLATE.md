# 기초파일 양식 설명

대시보드에서 `기초파일 양식 다운로드` 버튼을 누르면 아래 구조의 엑셀 파일을 받을 수 있습니다.

## 기본 원칙

- 한 행은 상품 또는 옵션 하나를 의미합니다.
- 옵션별로 관리하지 않을 상품은 `옵션명`을 비워도 됩니다.
- 원본 업로드 파일은 서버에 계속 보관됩니다.
- `현재고`, `최근한주수량`, `총판매수량`만 있어도 최근한주 기준 판매가능주는 계산됩니다.
- 총판매 기준 판매가능주를 정확히 계산하려면 `오픈일` 또는 `판매일수`가 필요합니다.
- 입고후 재고를 계산하려면 `입고예정수량`이 필요합니다.

## 컬럼 설명

| 컬럼명 | 필수 여부 | 예시 | 설명 |
|---|---:|---|---|
| 상품코드 | 선택 | P001 | 내부 상품코드, 공급처 옵션 코드, 이카운트 코드 등을 넣습니다. 없으면 비워도 됩니다. |
| 상품명 | 필수 | 촤르르반팔 | 대시보드에서 품목별 요약 기준으로 사용됩니다. |
| 옵션명 | 선택 | 블랙/M | 옵션별로 보고 싶을 때 입력합니다. 옵션 관리가 필요 없으면 비워도 됩니다. |
| 현재고 | 필수 | 30 | 현재 판매 가능한 재고입니다. 기존 파일의 가용재고/현재고에 해당합니다. |
| 최근한주수량 | 필수 | 10 | 최근 한 주 동안 판매된 수량입니다. |
| 총판매수량 | 필수 | 100 | 오픈일부터 현재까지 누적 판매수량입니다. |
| 오픈일 | 권장 | 2026-06-01 | 총판매 기준 판매가능주 계산에 사용합니다. `판매일수`가 있으면 생략 가능하지만, 둘 중 하나는 권장합니다. |
| 판매일수 | 권장 | 30 | 총판매수량이 몇 일 동안 쌓인 수량인지 의미합니다. `오픈일`이 있으면 생략 가능합니다. |
| 입고예정일 | 선택 | 2026-06-19 | 입고가 예정된 날짜입니다. 분할 입고가 되면 변경된 일정으로 다시 업로드하면 새 일정이 보관됩니다. |
| 입고예정수량 | 선택 | 50 | 앞으로 입고될 수량입니다. 이 값을 넣어야 입고후재고가 계산됩니다. |
| 배송수량 | 선택 | 0 | 일자별 배송수량을 관리하고 싶을 때 입력합니다. |
| 접수수량 | 선택 | 0 | 미출고/접수 수량을 관리하고 싶을 때 입력합니다. |

## 계산 방식

```text
입고후재고 = 현재고 + 입고예정수량
현재고 기준 최근한주 판매가능주 = 현재고 / 최근한주수량
입고후 기준 최근한주 판매가능주 = 입고후재고 / 최근한주수량
총판매 기준 주평균 판매량 = 총판매수량 / 판매일수 * 7
현재고 기준 총판매 판매가능주 = 현재고 / 총판매 기준 주평균 판매량
입고후 기준 총판매 판매가능주 = 입고후재고 / 총판매 기준 주평균 판매량
```

## 최소 입력 예시

| 상품명 | 현재고 | 최근한주수량 | 총판매수량 |
|---|---:|---:|---:|
| 촤르르반팔 | 30 | 10 | 100 |
| 냉감이불 | 80 | 20 | 300 |

## 입고예정까지 쓰는 예시

| 상품명 | 옵션명 | 현재고 | 최근한주수량 | 총판매수량 | 오픈일 | 입고예정일 | 입고예정수량 |
|---|---|---:|---:|---:|---|---|---:|
| 촤르르반팔 | 블랙/M | 30 | 10 | 100 | 2026-06-01 | 2026-06-19 | 50 |
| 냉감이불 | 화이트/Q | 80 | 20 | 300 | 2026-05-01 |  | 0 |


## 과거 배송과 입고예정 변경 관리

- 월/수/금처럼 같은 주에 여러 번 업로드해도 업로드 원본은 계속 보관됩니다.
- 업로드 화면의 `작성/기준일`에 `2026-06-15`, `2026-06-17`, `2026-06-19`처럼 입력하면 배송수량은 해당 날짜의 배송 기록으로 따로 저장됩니다.
- 따라서 6/17 파일을 올려도 6/15 배송 기록은 사라지지 않습니다.
- 입고예정은 확정 재고가 아니라 일정입니다. 6/19 입고 예정이 6/19와 6/25로 나뉘면, 다음 업로드 파일에 변경된 입고예정일/입고예정수량을 행으로 넣으면 됩니다.
- 이후에는 입고 일정 전용 입력 화면을 추가해서 엑셀을 다시 올리지 않고 웹에서 바로 수정할 수 있게 만드는 것이 다음 단계입니다.


## 분리 다운로드 양식

대시보드에는 아래 양식 다운로드 버튼이 있습니다.

### 1. 현재고양식

```text
상품코드 / 공급처옵션명 / 상품명 / 옵션 / 가용재고 / 접수 / 송장 / 배송 / 송장+배송
```

### 2. 최근한주판매수량 양식

```text
상품코드 / 공급처옵션명 / 상품명 / 옵션 / 수량
```

### 3. 총판매수량 양식

```text
상품코드 / 공급처옵션명 / 상품명 / 옵션 / 수량
```

### 4. 상품기본정보/오픈일 양식

```text
상품코드 / 공급처옵션명 / 상품명 / 옵션 / 오픈일
```

오픈일은 매주 바뀌는 값이 아니므로 `상품기본정보/오픈일 파일 업로드` 방식으로 한 번 등록해두고 계속 재사용하는 구조가 맞습니다. 이후 현재고/최근한주/총판매 데이터를 업로드하면 저장된 오픈일을 찾아 총판매 기준 판매가능주 계산에 사용합니다.
